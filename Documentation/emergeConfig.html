About

emergeConfig is a configuration application for Emerge Desktop. It allows for the changing of the various settings of each individual applet.

Usage

The first page lists the version of emergeCore and emergeLib you are using as well as the applets that are configured to execute at astart up. On this page you can start and stop each of the listed applets.

On the left-hand side of the window you will see the various catagories for which each applet falls into. Expanding a category will list all the appropriate applets.

Each applet has it's own configuration page with the appropriate settings which you can modify. Clicking apply will apply the changes immediately.

The emergeCore tab allows for the definition of the menu. One of the available menu item types is a registry based submenu. To add a registry based submenu, do the following:

  1. Choose 'New', and 'Submenu' in the 'Type' combobox
  2. Enter a caption for the submenu, starting with '-' (i.e.: -mysubmenu)
  3. Click on 'Save'
  4. Choose 'New' again, and whatever you want in 'Type'
  5. Enter a caption for the submenu item, starting with '--' (i.e.: --my item nr.1)
  6. Click on 'Save'
  7. Repeat steps 4 to 6 for all the entries
  8. When you are finished, click the 'Apply' button to apply the changes
Shell Changer

emergeConfig included a Shell Changer for switching between Emerge Desktop and Explorer (or any other shell you choose to add). Shell Changer supports the following options:

Current User Only

Note: The current user installation is based on the tutorial by Björn Brembs (http://brembs.net/litestep/win2k.html).

Check this option to enable "Per User" setting. The Shell path will be stored in HKCU section of the registry, and the following HKLM key:

Software\Microsoft\Windows NT\CurrentVersion\IniFileMapping\system.ini\boot\Shell

Will be set to:

USR:Software\Microsoft\Windows NT\CurrentVersion\Winlogon

Which tells Windows to load the Shell defined in the HKCU section. This will allow each user to use a different Shell. If this item is not checked, the Shell path will be stored in HKLM section of the registry, and the above HKLM key will be set to:

SYS:Software\Microsoft\Windows NT\CurrentVersion\Winlogon

Which tells Windows to load the Shell defined in HKLM section. This will set the choosen Shell as the system shell for all users.

Enable "DesktopProcess"

Check this option if you need to run occasionally explorer.exe for file management (i.e., if you need to use TortoiseCvs, or some other tool wich needs explorer to run properly). The setting will be stored in HKCU or HKLM section of the registry, depending on the selection done for "Current User Only" option.

The following key (in HKCU or HKLM) will be set to "1" (true):

Software\Microsoft\Windows\CurrentVersion\Explorer\DesktopProcess

Enable "BrowseNewProcess"

Check this option if you need to run more than one explorer.exe window at the same time. The setting will be stored in HKCU or HKLM section of the registry, depending on the selection done for "Current User Only" option. The following key (in HKCU or HKLM) will be set to "yes":

Software\Microsoft\Windows\CurrentVersion\Explorer\BrowseNewProcess\BrowseNewProcess

Set Shell button

This is the button used to set a Shell, that is, write its Path to the appropriate registry section. Depending on the state of the "Current User Only" checkbox, the following key:

Software\Microsoft\Windows NT\CurrentVersion\Winlogon\Shell

in HKCU or HKLM will be set to the currently selected Shell Path in the Shell List Window. The Shell Path must always be an "absolute path", with the only exception of explorer.exe.

You can use whatever you like for the "Shell Name" field.

General Settings

This page contains options that apply to all applets. The options include:

  • Disable "Drop Shadow" for all applets - globally disable drop shadows for all applets, overriding the individual applet setting
  • Enable "Drop Shadow" for all applets - globally enable drop shadows for all applets, overriding the individual applet setting
  • Enable custom "Drop Shadow" for each applet - allows for the enabling of the "Drop Shadow" on a per applet basis

Note: The "Drop Shadow" effect required Windows XP or above

Theme Manager

Finally, the Theme Manager allows for the storage of a given set of applet configurations (a theme). This allows for the setting of various configurations that can be switched with the click of a button!

Themes are actually emergeScripts, as such, they are required to have the '.esc' extension. They must be stored in '..\Emerge Desktop\themes'. For details on how to create and modify theme scripts, see the emergeScriptEdit documentation.

Theme manager buttons include:

  • restart: restart Emerge Desktop (note: currently emergeTray does not handle the restart gracefully, this will be corrected in a later release).
  • refresh: reload the theme list
  • edit: open the selected theme with emergeScriptEdit
  • load (or double clicking a list item): execute the selected theme script
Note

In order for emergeConfig to operate properly the proper directory structure must be used.

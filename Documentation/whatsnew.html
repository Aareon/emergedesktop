v4.1 (April 18th, 2008)

New:
  • More EmergeVars:
    • %Desktop% - current user's desktop
    • %CommonDesktop% - All Users' desktop
    • %Documents% - current user's "My Documents"
    • %CommonDocuments% - All Users' "My Documents"
    • Like the %StartMenu% and %CommonStartMenu%, you can use these as "Executable" to pop them open in a file manager, or as a "Folder Menu" to open them as menus. With the "Folder Menu", you can merge them, (i.e. %Desktop%|%CommonDesktop%, etc.)
  • "Center" alignment for both vertical and horizontal. The applets will "grow" from the center if autosize is enabled, if not they align themselves to the center of the applet window.
  • Multi-line support in emergeCommand (for time and tip). To introduce a "newline" use "%_" (i.e. %A%_%x%_%X).
  • CPL files now display their icon correctly
  • emergeVWM now "loops" when you use hotkeys to change the desktop (courtesy of Teshadael)
  • You can now use the "Windows" key only as a hotkey (like Explorer with the start menu). Hotkeys trigger on key-up now (as opposed to key-down) to allow for the "Windows" key to be used for multiple hotkeys.
  • Internal commands "VolumeUp", "VolumeDown" and "VolumeMute" Internal Commands
  • emerge.exe utility to allow non-Emerge Desktop applications to execute Internal Commands
Changed:
  • Improved fullscreen check so the issues with VLC et. all should now have gone away
  • Fixed setting the shell under x64 windows
  • Fixed emergeVWM so that mini windows do not remain in the "drag" state if the left mouse button is released off of the emergeVWM window
  • Control Panel should show all the icons under x64 Windows
  • emergeHotkeys only allows one instance of it's configuration dialog.
  • emergeHotkeys configuration dialog doesn't reset if the key combination fails to register.
  • emergeCore now adds quotes around the shell when setting it, if the shell path has spaces in it.
  • Fixed bug where the tasks menu (i.e. the right click menu on an icon in emergeTasks) was displaying when it shouldn't (i.e. the options dialog of firefox is open and your right click on the firefox icon in emergeTasks)
  • Installer will now restart emergeCore if it was running prior to the installer running
Removed:
  • "My Documents" Special folder removed in favor of the %Documents%, %CommonDocuments% EmergeVars

v2.0.1 (Sep 9th, 2005) - A Settings entry is included in the default Right menu. A shortcut is added to the Emerge group to allow to easily switch between shells.

The emergescript language is dropped. The Right and Mid menus have an Editor which supports drag and drop sorting and has a tree structure. Most settings can be accessed via internal commands. Each applet has now it's own icon created by moeghan.

The core applets support editable schemes which allow to set colour and transparency of each applet individually. Tray and tasks can now autoresize.The default midmenu has now some essential commands including access to Current User and All User Programs.

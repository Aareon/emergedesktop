Directory Structure

In order for Emerge Desktop to operate properly, the following directory structure must be used:

Emerge Desktop 
|
|------ documentation
|   |
|   |------ Emerge Desktop.chm
|
|------ emergeAppletEngine.dll
|------ emergeBaseClasses.dll
|------ emergeCore.exe
|------ emergeDesktop.exe
|------ emergeGraphics.dll
|------ emergeIcons.dll
|------ emergeLib.dll
|------ emergeSchemeEngine.dll
|------ emergeTasks.exe
|------ emergeTray.exe
|------ libgcc_s_dw2-1.dll
|------ libstdc++-6.dll
|------ all other Emerge Desktop applets

All settings (by default) are stored in %AppData%\Emerge Desktop\ under the following structure:

Emerge Desktop
|------ files
|   |
|   |------ cmd.txt (alias list for emergeCommand)
|   |
|   |------ (.xml files with user settings, namely emergeCommand history, emergeCore other shells,
|           emergeDesktop menu items, all emergeHotkey commands, emergeTray hide list and 
|           emergeVWM sticky list)
|
|------ scripts
|   |
|   |------ (scripts files in Perl, Python and Ruby that can be used to control Emerge Desktop)
|
|------ themes
|   |
|   |------ GBRY
|   |   |
|   |   |------ schemes
|   |   |   |
|   |   |   |------ (.eds scheme files for customizing applets)
|   |   |
|   |   |------ (.xml files with applet settings)
|   |
|   |------ Other themes
|       |
|       |------ schemes
|       |   |
|       |   |------ (.eds scheme files for customizing applets)|   |   |
|       |
|       |------ media
|       |   |
|       |   |------ (optional folder, ideal location for external icons, wallpaper, fonts)
|       |
|       |------ (.xml files with applet settings)

|
|------ theme.xml (This file just stores the name of the current Theme)

Although the structure must follow this organization, the location of the folders can be modified in order to allow Emerge Desktop to run in different Modes
